## What it is
MT (Manual Test) is a testing approach where test cases are executed manually by a
tester, without automated scripts driving the test steps.

## When to use it
Used for scenarios that are exploratory, low-frequency, or not yet worth automating —
e.g. one-off validation, early-stage feature testing, or edge cases that are hard to
script reliably.

## Key features
- No automation framework required
- Flexible — a human tester can adapt on the fly
- Lower setup effort compared to SAF/FAF

## Requirements / prerequisites
- A tester with domain knowledge
- Access to the test environment
- Manual test case documentation / checklist

## How it differs from SAF and FAF
MT has no automation layer, unlike SAF (partial automation) and FAF (full automation).
It's typically the starting point before a workflow graduates to SAF or FAF.
