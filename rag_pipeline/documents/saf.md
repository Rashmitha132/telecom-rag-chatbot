## What it is
SAF (Sanity Automation Framework) is an automated framework focused on quick sanity
checks — fast, lightweight tests that confirm core functionality works before deeper
testing begins.

## When to use it
Used after a build/deployment to quickly verify nothing critical is broken, before
committing to a full test cycle.

## Key features
- Fast execution (sanity-level, not exhaustive)
- Automated — reduces manual effort vs MT
- Lighter weight than FAF

## Requirements / prerequisites
- Automation scripts/framework set up
- Test environment access
- CI/CD or trigger mechanism (if run automatically after builds)

## How it differs from MT and FAF
Unlike MT, SAF is automated. Unlike FAF, SAF covers only sanity-level checks rather
than the full test suite — it's the middle ground between manual testing and full
automation.
